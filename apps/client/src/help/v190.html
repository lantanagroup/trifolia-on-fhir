v1.9.0

Release 1.9.0

Specify an IG Jurisdiction

You can now add Jurisdictions to your IG.

IGs with no specified jurisdiction may result in this warning from the publisher:

ImplementationGuide.jurisdiction warning The resource should declare its jurisdiction to match the package id (hl7.fhir.us.ccda, jurisdiction = urn:iso:std:iso:3166:-2#US)

Package IDs that contains "hl7.fhir.us" populate the Jurisdiction fields with a CodeableConcept. The system is "urn:iso:std:iso:3166:-2" and the code is "US."

Select the IG Publisher Version when Exporting/publishing


On the Publish screen, you can now select which version of the IG publisher to run. The server uses the IG publisher version to run the publishing process.


Resolve Common Pre-Publishing Errors

If you encounter errors during the publishing process, check chat.fhir.org for guidance.


The new Publishing section in the Help Documentation documents steps how to resolve common pre-publishing errors encountered with the current version:

  • IG Version
  • Generating Snapshot
  • IG Package ID
  • Terminology Server Connection
  • Dependency
